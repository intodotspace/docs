Space는 사용자를 보호하고 마켓 무결성을 유지하기 위해 여러 층의 리스크 통제를 구현합니다.
## 오라클 보안
**다중 소스 검증:**
- [Pyth](https://www.pyth.network/) + [Switchboard](https://switchboard.xyz/) 가격의 중앙값
- [오라클](/ko/resolution/oracle) 간 차이가 >2%이면 거부
- 15분 TWAP로 조작 평균화
**활성 검사:**
- TWAP 윈도우 내 최소 업데이트 요구
- 오라클이 일시적으로 다운되면 윈도우 연장
- 데이터 부족 시 무효 표시
**가격 제한:**
- 분당 최대 1% 가격 변동
- 이상치 감지: >3 표준편차 가격 거부
- 다중 피드 중앙값으로 단일 소스 조작 방지

## 이벤트 전 동결
마켓은 결산 전에 동결 메커니즘을 구현할 수 있습니다:
**옵션 1: 수수료 인상**
결산까지 시간 < 10분: $\text{Taker 수수료} \times 3$
결산까지 시간 < 5분: $\text{Taker 수수료} \times 5$
**옵션 2: 포지션 동결**
결산까지 시간 < 5분:
- 신규 포지션: 비활성화
- 포지션 청산: 활성화
내부 정보나 빠른 정보 접근으로 인한 불공정한 이점을 이용한 막판 조작을 방지합니다.
## 슬리피지 보호
사용자는 시장가 주문에 대해 최대 허용 슬리피지를 설정합니다(기본값 5%).
$\frac{|\text{실제 체결 가격} - \text{예상 가격}|}{ \text{예상 가격}} > \text{최대 슬리피지}$인 경우: → 트랜잭션 되돌림 → 거래 미체결 → 수수료 미청구
변동성이 크거나 유동성이 낮은 마켓에서 예상치 못한 체결 가격으로부터 사용자를 보호합니다.
## 플랫폼 업그레이드
**타임락 요구사항:**
- 파라미터 변경: 48시간 타임락
- 중요 코드 변경: 72시간 타임락
**멀티시그 제어:**
- 관리 기능: 3/5 멀티시그
- 결산 제출: 2/3 멀티시그
- 긴급 일시 중지: 2/5 멀티시그 (빠른 대응)
모든 거버넌스 작업은 온체인에서 투명하게 기록됩니다.
## 미래: zkML 및 zkTLS 연구
Space는 예측 마켓에서 아직 활용되지 않은 더욱 최첨단의 신뢰 불필요한 주관적 결산을 위한 [영지식 증명](/ko/concepts/resolution)을 연구하고 있습니다:
**zkML (영지식 머신러닝):**
- 모델 가중치를 공개하지 않고 AI 모델 출력 증명
- 결산 로직의 온체인 검증
- 결산 운영자에 대한 신뢰 불필요
**zkTLS (영지식 전송 계층 보안):**
- 특정 URL에서 데이터를 가져왔음을 증명
- 출처 진위성의 암호학적 증명
- zkML과 결합하여 완전 자동화된 결산
**결합 아키텍처:** zkTLS를 통해 뉴스 기사 가져오기 → zkML로 처리 → 온체인에 증명 제출 → 암호학적 보장이 있는 자동 결산.
