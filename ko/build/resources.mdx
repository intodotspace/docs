## 시작하기
<Steps>
  <Step title="API 탐색" titleSize="h3">
    공개 마켓 데이터는 인증이 필요 없습니다. 실험 시작:
    - `GET /markets` - 사용 가능한 마켓 조회
    - `GET /markets/{id}` - 상세 마켓 정보 조회
    - `GET /markets/{id}/orderbook` - 현재 유동성 확인
    - 실시간 피드를 위한 **WebSocket** 연결 테스트
  </Step>
  <Step title="로컬 개발" titleSize="h3">
    플랫폼 개발 중:
    - 테스트용 모의 마켓 데이터 사용
    - 통합 아키텍처 설계
    - 데이터 흐름 및 오류 처리 계획
    - API 출시 준비
  </Step>
  <Step title="실서비스 전환" titleSize="h3">
    프로덕션 API 출시 후:
    - 라이브 엔드포인트에 연결
    - 응답 헤더의 **요청 제한** 모니터링
    - 적절한 오류 처리 구현
    - 통합 배포
  </Step>
</Steps>
## 모범 사례
**캐싱**
자주 변경되지 않는 데이터 캐시:
- 마켓 메타데이터 (질문, 만료일, 카테고리)
- 역사적 거래 데이터
**정기적으로 갱신:**
- 현재 가격 - **5-10초**마다
- 오더북 깊이 - **WebSocket을 통한 실시간**
- 거래량 통계 - **매분**
**WebSocket 재연결** 네트워크는 실패합니다. 통합이 이를 처리해야 합니다:
- **자동 재연결** 구현
- **지수 백오프** 사용 (1s, 2s, 4s, 8s…)
- 재연결 후 채널 **재구독**
- 알림 전 **최대 재시도 횟수** 고려
## 오류 처리
스마트 재시도 로직:
- **5xx 오류** - 백오프로 재시도 (서버 문제, 일시적일 가능성)
- **4xx 오류** - 재시도하지 않음 (클라이언트 오류, 요청 수정 필요)
- **429 (요청 제한)** - `Retry-After` 헤더 준수 후 재시도
- **네트워크 오류** - 백오프로 재시도
## 요청 제한 관리
제한 이내 유지:
- 로컬에서 분당 요청 수 추적
- `X-RateLimit-Remaining` 헤더 확인
- 제한에 도달하기 전에 백오프
- 폴링 대신 실시간 데이터용 **WebSocket** 사용
