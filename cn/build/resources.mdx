## 入门指南
<Steps>
  <Step title="探索 API" titleSize="h3">
    公开市场数据无需认证。开始体验：
    - `GET /markets` - 查看可用市场
    - `GET /markets/{id}` - 获取详细市场信息
    - `GET /markets/{id}/orderbook` - 查看当前流动性
    - 测试 **WebSocket** 连接获取实时数据流
  </Step>
  <Step title="本地开发" titleSize="h3">
    平台开发期间：
    - 使用模拟市场数据进行测试
    - 设计您的集成架构
    - 规划数据流和错误处理
    - 为 API 上线做准备
  </Step>
  <Step title="上线" titleSize="h3">
    生产 API 上线后：
    - 连接到生产端点
    - 监控响应头中的**速率限制**
    - 实现正确的错误处理
    - 部署您的集成
  </Step>
</Steps>
## 最佳实践
**缓存**
缓存不经常变化的数据：
- 市场元数据（问题、到期时间、类别）
- 历史交易数据
**定期刷新：**
- 当前价格 - 每 **5-10 秒**
- 订单簿深度 - **通过 WebSocket 实时获取**
- 交易量统计 - **每分钟**
**WebSocket 重连** 网络会出故障。您的集成应该能够处理：
- 实现**自动重连**
- 使用**指数退避**（1s、2s、4s、8s…）
- 重连后**重新订阅**频道
- 考虑在告警前设置**最大重试次数**
## 错误处理
智能重试逻辑：
- **5xx 错误** - 使用退避重试（服务器问题，可能是暂时的）
- **4xx 错误** - 不要重试（客户端错误，修复您的请求）
- **429（速率限制）** - 遵守 `Retry-After` 头，然后重试
- **网络错误** - 使用退避重试
## 速率限制管理
保持在限制以下：
- 在本地跟踪每分钟请求数
- 检查 `X-RateLimit-Remaining` 头
- 在达到限制前退避
- 使用 **WebSocket** 获取实时数据而不是轮询
